# Governance Documentation

The project is currently owner-directed. Governance should remain lightweight until there is a useful implementation and active contributors.

Future documents include:

- Decision-making process
- Contribution guide
- Code of conduct
- Security policy and vulnerability reporting
- Licensing rationale
- Maintainer responsibilities
- Plugin review principles

Major product scope, principle, security-risk, and architecture decisions require explicit approval from the project owner during the preview.
