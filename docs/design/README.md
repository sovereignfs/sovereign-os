# Design Documentation

Design work describes user problems, flows, states, accessibility, and trust communication before interface implementation.

## Preview Design Backlog

- First-run onboarding
- Dashboard and system health
- Chat and capability activity
- Pi-hole connection settings
- Loading, offline, unauthorized, unsupported, and failed states
- Local versus remote AI disclosure
- Responsive phone and desktop behavior
